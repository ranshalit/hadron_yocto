# meta-hadron

Yocto BSP layer for the ConnectTech **Hadron NGX012** (Jetson Orin Nano 4GB / p3767-0003 module).

Built on top of [meta-tegra](https://github.com/OE4T/meta-tegra) scarthgap branch targeting **L4T 36.5.0 / JetPack 6.2.2**.

## Features

| Feature | Details |
|---------|---------|
| Board | ConnectTech Hadron NGX012 |
| Module | Jetson Orin Nano 4GB (p3767-0003) |
| Kernel | linux-jammy-nvidia-tegra 5.15 |
| DTB | CTI full-board DTB (`tegra234-orin-nano-cti-NGX012.dtb`) |
| IMU | BMI160 over I2C (`kernel-module-bmi160-core`, `kernel-module-bmi160-i2c`) |
| Network | eth0 static IP `192.168.132.100/24` |
| User | `ubuntu` / `ubuntu` (sudo) |
| Init | systemd |

## Quick Start

### Prerequisites

```bash
pip3 install kas
sudo apt-get install -y gawk wget git diffstat unzip texinfo gcc build-essential \
    chrpath socat cpio python3 python3-pip python3-pexpect xz-utils debianutils \
    iputils-ping python3-git python3-jinja2 libegl1-mesa libsdl1.2-dev \
    python3-subunit mesa-common-dev zstd liblz4-tool file locales libacl1
```

### Build

```bash
git clone <this-repo> hadron_yocto
cd hadron_yocto
kas build kas.yml
```

The tegraflash deployment package will be at:
```
build/tmp/deploy/images/hadron-ngx012/hadron-image-base-hadron-ngx012.rootfs.tegraflash.tar.gz
```

### Flash

Put the board in **USB recovery mode** (hold RECOVERY button while applying power), then:

```bash
tar -xzf build/tmp/deploy/images/hadron-ngx012/hadron-image-base-hadron-ngx012.rootfs.tegraflash.tar.gz
cd hadron-image-base-hadron-ngx012.rootfs.tegraflash/
sudo ./doflash.sh
```

Verify the board returns to normal boot and is reachable at `192.168.132.100`.


## Known Issues / Notes

- `CONFIG_MODVERSIONS=n` is forced: binutils ≥ 2.38 on AArch64 rejects `R_AARCH64_ABS32` relocations generated by MODVERSIONS. Safe in a Yocto image where all modules come from the same build.
- CTI BSP tarball used: `CTI-L4T-ORIN-NX-NANO-36.4.4-V005.tgz` (L4T 36.4.4 artifacts in an L4T 36.5.0 Yocto build — minor point-release, tested compatible).
